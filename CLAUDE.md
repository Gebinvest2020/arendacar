# Правила работы над проектом Arendacar

Это правила для ассистента (Claude) при работе с этим проектом.

## Общение

- Всегда отвечать на **русском языке**.
- Объяснять результат простыми словами, понятными без глубоких технических знаний.

## Git

- **Перед любыми изменениями** проверять состояние репозитория командой `git status`.
- **Не использовать** `git add .` — это добавляет всё подряд.
- Добавлять файлы в Git **только явными путями** (например: `git add src/app/page.tsx`).
- **Не делать `git commit` и `git push` без явного разрешения пользователя.**

## Безопасность

- **Не добавлять** в проект реальные секреты, ключи, пароли и токены.
- Секретные значения хранить только в `.env*` файлах (они в `.gitignore`) и не коммитить.

## Проверка кода

- После любых изменений запускать:
  - `npm run lint`
  - `npm run build`
- Если появились ошибки — исправлять их и объяснять, что было не так.

## Границы проекта

- Это отдельный проект, **не связанный** с broker-core. Не трогать чужие папки на диске.
- Пока **не подключать**: базу данных, Prisma, авторизацию, админку, оплату, Telegram, API бронирования.
- Сайты `catcar.ua` и `tnd.com.ua` использовать **только как референс по структуре и функциям** — не копировать их дизайн, код, тексты, логотипы и изображения.
